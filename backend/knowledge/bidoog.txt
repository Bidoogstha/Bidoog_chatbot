### PROJECT 1: MindfulCart
GitHub: github.com/moona-dance/mindfulcart
Hackathon: Won 1st place (FinTech track) at AugieHack 2026 at Augustana College
Team: Built with Aigerim Nussupova and Altynai Ruslanova in 24 hours
Category: FinTech / Behavioral Economics / Chrome Extension

What it does:
MindfulCart is a Chrome extension with a companion web app that intercepts users at the moment of online checkout and adds a 30-second conscious pause before the purchase goes through. It is not designed to stop purchases — only to make them intentional rather than reflexive. Online checkout flows are engineered to minimize friction and maximize impulse buying. MindfulCart adds one small moment of friction back in.

How it works technically:
- The Chrome extension (Manifest V3) detects when a user is on a checkout page on Amazon or other shopping sites
- It quietly reads the item name and price from the product page in the background
- When the user proceeds to checkout, the extension reads that saved data and passes it to the web app via URL parameters
- The web app asks the user a few questions about their mood and whether they genuinely need the item
- It shows how many hours of work the purchase costs in real terms
- The user sees a Canvas API-drawn visualization of their spending patterns
- Data is stored in localStorage — nothing is sent to external servers
- The Claude/Anthropic API was integrated to generate personalized purchase summaries, mood-based decision prompts, and spending insights

Technical stack:
- Frontend: React + Vite
- Backend: Express.js (Node.js) running on port 3001
- Extension: Chrome Manifest V3
- Charts: Raw Canvas API — no chart library used
- AI: Anthropic Claude API for real-time purchase summaries
- Data storage: localStorage (privacy-first, nothing leaves the browser)

Bidoog's role:
- Led backend development
- Integrated the Claude API
- Designed prompts and system instructions for personalized recommendations and mood-based decision prompts
- Built spending visualizations using the Canvas API

Why it won:
The judges were impressed by the behavioral economics angle — using a 30-second pause to interrupt the impulse buying loop — combined with the technical execution of a full Chrome extension, web app, and AI integration within 24 hours.

### PROJECT 2: Loan Approval ML — End-to-End Binary Classification
GitHub: github.com/Bidoogstha/loan-approval-ml
Live demo: https://loan-approval-ml-bidoog.streamlit.app/
Timeline: March 2026 – Present

What it does:
An end-to-end binary classification system that predicts loan default risk on 1.3 million real Lending Club loans (2007–2018). Built to demonstrate how a real lending team would approach the problem — not the way most online tutorials do.

Key technical decisions and why:
- Real data, not synthetic: ~1.3M loans from Kaggle Lending Club dataset, filtered to applications with known outcomes (Fully Paid or Charged Off)
- The pipeline cleans 150+ raw columns down to 15 predictive features
- Post-origination features (total_pymnt, recoveries, etc.) are deliberately excluded to prevent target leakage — a trap most beginner notebooks fall into
- Engineered features: loan-to-income ratio, installment-to-income ratio

Five model families compared:
1. Logistic Regression — interpretable baseline
2. Random Forest — strong out-of-the-box, handles non-linearities
3. XGBoost — state-of-the-art for tabular data
4. LightGBM — faster than XGBoost, often comparable
5. Stacking Ensemble — Logistic Regression meta-learner over RF + XGB + LGBM

Each model tuned with Optuna (TPE sampler, 50 trials) optimizing 5-fold cross-validated ROC-AUC.
Best ROC-AUC: approximately 0.74 — in line with published Lending Club benchmarks.

Fairness Analysis (using Fairlearn):
Three mathematically distinct fairness metrics measured across two sensitive attributes (US state and income bracket):
- Demographic parity: PASS — 7.7pp disparity, ratio 0.92 (above four-fifths rule threshold)
- Calibration by group: FAIL — 11.0pp disparity by state
- Equalized odds: FAIL — 28-percentage-point TPR gap across US states

What the 28pp gap means: The model is much better at correctly identifying creditworthy borrowers in some states than others. When it predicts 20% default risk, the actual rate in some states is 10% and in others 30%. This is the kind of finding any deployed lending model would face in a fair-lending review. Resolving it would require group-conditional thresholds, fairness-constrained training, or different training data.

This empirically demonstrates the impossibility result from Pleiss et al. (NeurIPS 2017): a model cannot satisfy demographic parity, equalized odds, and calibration simultaneously unless perfectly accurate or base rates are equal.

Explainability:
- SHAP TreeExplainer for global feature importance and per-prediction waterfall plots
- Permutation importance and partial dependence plots
- Interactive What-if explorer in the Streamlit app

Streamlit App (4 tabs):
1. Predict — real-time risk prediction with SHAP waterfall
2. Models — comparison across all five model families
3. Data — EDA and feature distributions
4. Fairness — per-state and per-income-bracket disparity breakdown

Testing: 16 passing pytest tests
Limitations honestly documented: no temporal validation yet (random split rather than time-based), equalized odds gap unresolved, threshold 0.5 is rarely optimal in production.

References used: SHAP (Lundberg & Lee 2017), Optuna (Akiba et al. 2019), XGBoost (Chen & Guestrin 2016), LightGBM (Ke et al. 2017), Fairlearn, Hardt et al. 2016 (equalized odds), Kleinberg et al. 2017 (impossibility), Griggs v. Duke Power 1971, CFPB fair lending guidelines.
